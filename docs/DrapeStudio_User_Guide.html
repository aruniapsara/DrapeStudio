1. What is DrapeStudio?

DrapeStudio is an AI-powered garment model image generator designed for clothing sellers. You upload photos of your garment, configure a virtual model and scene, and receive photorealistic catalogue-quality images — no photoshoot required.

Images are generated using Google's Gemini AI image generation model. Each generation produces 3 images (adult/children modules) or 2 images (accessories module), each from a different camera angle.

Key Features

  • 4 Modules: Adult Fashion, Children's Fashion, Accessories, Virtual Fit-On
  • Customisable Models: Gender, age, ethnicity, body type, hair, skin tone
  • Scene Control: Environment, pose, framing, and lighting
  • Multi-language: English, Sinhala, Hindi, Tamil
  • History & Reuse: Past generations and uploaded images are saved
  • Download: Individual images or download all as a ZIP file

4. Module: Adult Fashion

Step 1 — Upload Garment Photos

1
Select a garment type from the category cards: Casual, Formal, Saree, Shalwar Kameez, Batik, or Sarong.
2
Choose the model gender: Feminine, Masculine, or Unisex.
3
Upload 1–5 garment photos by dragging and dropping, clicking the upload area, or using the camera (mobile). Accepted formats: JPG, PNG, WEBP. Minimum 400×400 px.
4
(Optional) Load from Previous Uploads — expand the history section to reuse previously uploaded garment images without re-uploading.
Info: Images larger than 5 MB are automatically compressed. You can remove individual photos by clicking the × button on each thumbnail.

Step 3 — Review

Review all your settings before generating:

  • Garment photos (scrollable thumbnails)
  • Model appearance summary
  • Scene settings summary
  • Measurements (if provided)
  • Credit cost estimate and remaining credits

Click "Generate Images" to start.

Step 4 — Generating

A polling screen shows real-time status updates:

  • Queued — Waiting for a worker
  • Running — AI is generating your images
  • Succeeded — Redirects to Results
  • Failed — Shows error message (credits not deducted)

Generation typically takes 30–90 seconds depending on complexity.

Note: Do not close the browser tab during generation. The page polls the server every 1–2 seconds for status updates.

Step 5 — Results

View your generated images:

  • 3 images from different camera angles: Front view, Three-quarter view, Back view
  • Navigate between images using arrow buttons or swipe (mobile)
  • Generation cost displayed (e.g. "$0.003")
  • Download: Individual image or "Download All as ZIP"
  • Try Again: Re-generate with the same settings
  • New Upload: Start a completely new generation

7. Module: Virtual Fit-On

Access via the "Virtual Fit-On" card on the home page, or navigate to /fiton.

How It Works

1
Upload Garment: Upload garment photos, select garment type, optionally provide garment measurements (size label, chest, waist, hips, length).
2
Customer Details: Upload a customer photo (required), enter customer measurements (height, chest, waist, hips, shoulder width), and select fit preference (Regular, Tight, Loose, Oversized).
3
Results: View the virtual try-on image plus a size recommendation with fit confidence score (e.g. "Size M — 92% confident").
Info: The Fit-On module requires more credits than standard generation due to the additional sizing analysis.

12. Tester Checklist

Authentication

  • Google OAuth login (first-time user → profile onboarding)
  • Google OAuth login (returning user → goes to home)
  • Logout clears session and redirects to login
  • Unauthenticated access to /upload redirects to /login

Adult Module — Full Flow

  • Select garment type (e.g. Casual)
  • Select gender (Feminine)
  • Upload 1 garment photo → preview appears
  • Upload 5 garment photos (max) → 6th is rejected
  • Upload non-image file → error toast
  • Remove a photo from preview grid
  • Load image from "Previous Uploads" history
  • Configure: change age, ethnicity, body type, hair
  • Configure: select environment, pose, framing
  • Configure: Quick Preview card shows correct summary
  • Configure: garment thumbnail shows (not a broken icon)
  • Review: all settings displayed correctly
  • Review: credit cost shown
  • Generate: status transitions (queued → running → succeeded)
  • Results: 3 images displayed, gallery navigation works
  • Results: download individual image
  • Results: download all as ZIP
  • Results: "Try Again" regenerates with same config

Children's Module

  • Select each age group (Baby, Toddler, Kid, Teen)
  • Configure child-specific options (expression, pose)
  • Generate → 3 images with appropriate child model
  • Invalid age/pose combination is rejected by safety validator

Accessories Module

  • Select accessory category (e.g. Necklaces)
  • Select display mode (On-Model, Flat-Lay, Lifestyle)
  • Generate → 2 images with 1:1 aspect ratio
  • Flat-lay images show correct surface (marble, wood, etc.)

Virtual Fit-On

  • Upload garment + enter garment measurements
  • Upload customer photo + enter customer measurements
  • Results show virtual try-on image
  • Size recommendation with confidence score displayed

History & Management

  • History page lists past generations
  • Filter by module and status works
  • Pagination works
  • Delete a generation → removed from list
  • Download ZIP from history

Profile & Billing

  • Profile page shows correct user info
  • Edit display name → saves
  • Change language → UI updates
  • Free tier: 4th generation in a day is blocked
  • Credit balance displays correctly on review page

Error Handling

  • Generation fails → error message shown, credits NOT deducted
  • Network disconnect during polling → recovers on reconnect
  • Upload oversized file (>20 MB) → rejected
  • Upload undersized image (<400px) → warning shown

Mobile

  • Responsive layout on phone-sized screen
  • Camera capture works in upload step
  • Touch swipe works in results gallery
  • Buttons are tap-friendly (no tiny targets)

13. Troubleshooting

ProblemSolution
Login button does nothingCheck that Google OAuth is configured and the redirect URI matches.
Broken image icon on Configure pageThis was a known bug (fixed). Hard-refresh the page (Ctrl+Shift+R).
"GOOGLE_API_KEY is not configured"The server's .env file is missing the Google API key. Contact the admin.
Generation stuck on "Queued"The worker container may be down. Check with admin.
Generation fails immediatelyCheck the error message. Common causes: invalid API key, rate limit, or model not available.
Images look wrong or low qualityTry different environment/pose combinations. Use higher quality source photos (well-lit, plain background).
ZIP download is emptyOutput images may have expired (7-day retention). Generate again.
Language not changingClear browser cookies and log in again, then set language from Profile.
"Daily limit exceeded"Free tier allows 3 generations/day. Upgrade to Basic or Pro for more.
